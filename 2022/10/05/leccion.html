  1. GitHub Campus Expert
  2. Learning Markdown
  3. Setting up Our Environment
  • Veremos en que estado tiene cada uno el repo de la práctica github-campus-expert la solicitud del Student Developer Pack. Verifica la rúbrica de la práctica:
    • Rellenado cuestionario GitHub-Alu
    • El nombre del equipo es correcto
    • Solicitado Backpack
    • Kanban Board project conteniendo las incidencias en su estado
    • Convierto las notas del project en issues
    • Informe realizado mostrando que domina markdown de forma eficiente: Usa imágenes, enlaces, listas, etc.

  • Veremos en que estado tiene cada uno el repo de la práctica markdown la solicitud del Student Developer Pack. Verifica la rúbrica de la práctica:

    • Kanban Board project conteniendo las incidencias en su estado
    • Convierto las notas en issues
    • He generado el issue de práctica terminada
    • Informe elaborado correcto
    • Ha creado un repo profile en GitHub
    • Ha entregado el .zip en el campus con el repo
  • Veremos en que estado tiene cada uno el repo de la práctica setup la solicitud del Student Developer Pack. Verifica la rúbrica de la práctica:
    • SSH configurada en VSCode para trabajar sin password
    • git configurado en VSCode para trabajar con GitHub
    • Tutorial README.md bien presentado
    • El prompt de la shell en su máquina muestra la rama actual
    • Kanban Board project conteniendo las incidencias en su estado
    • Ha instalado nvm en su máquina
    • Ha entregado el .zip en el campus con el repo
